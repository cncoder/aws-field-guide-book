# 贡献指南

感谢你对Aurora PostgreSQL FDW Demo项目的关注！我们欢迎各种形式的贡献。

## 🤝 如何贡献

### 报告问题
- 使用GitHub Issues报告bug或提出功能请求
- 提供详细的问题描述和复现步骤
- 包含相关的错误日志和环境信息

### 提交代码
1. Fork这个仓库
2. 创建你的功能分支 (`git checkout -b feature/AmazingFeature`)
3. 提交你的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个Pull Request

### 改进文档
- 修正文档中的错误
- 添加更多使用示例
- 改进安装和配置说明
- 翻译文档到其他语言

## 📋 开发指南

### 环境要求
- AWS CLI 已配置
- Terraform >= 1.0
- 有效的AWS账户和权限

### 测试
在提交PR之前，请确保：
- [ ] Terraform配置可以成功部署
- [ ] 所有demo步骤都能正常执行
- [ ] 文档更新与代码变更保持同步
- [ ] 清理脚本能正确删除所有资源

### 代码规范
- 使用有意义的变量名和注释
- 遵循Terraform最佳实践
- 保持代码简洁和可读性
- 添加适当的验证和错误处理

## 🎯 贡献类型

### 🐛 Bug修复
- 修复Terraform配置错误
- 解决部署问题
- 修正文档错误

### ✨ 新功能
- 添加新的数据库配置选项
- 支持更多AWS区域
- 增加监控和告警功能
- 添加更多测试查询示例

### 📚 文档改进
- 添加故障排除指南
- 创建视频教程
- 改进架构图和说明
- 添加性能调优建议

### 🔧 基础设施改进
- 优化Terraform模块结构
- 添加自动化测试
- 改进CI/CD流程
- 增强安全配置

## 📝 提交信息规范

使用清晰的提交信息：
- `feat: 添加新功能`
- `fix: 修复bug`
- `docs: 更新文档`
- `refactor: 重构代码`
- `test: 添加测试`
- `chore: 维护任务`

## 🔍 代码审查

所有的Pull Request都会经过代码审查：
- 检查代码质量和最佳实践
- 验证功能是否按预期工作
- 确保文档完整和准确
- 测试在不同环境下的兼容性

## 📞 获取帮助

如果你需要帮助：
- 查看现有的Issues和Discussions
- 创建新的Issue描述你的问题
- 在Pull Request中@维护者
